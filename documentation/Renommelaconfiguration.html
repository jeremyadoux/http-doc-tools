Renomme la configuration

Utilisation de "Integration" ›› Détail des opérations ›› Configuration ››


Opération possible sur les pièces ou les assemblages.

Cette opération permet de renommer le nom d'une configuration.


Exemple : Renommer la configuration "Position Fixe" par "POSITION FIXE" du fichier parcouru.



Vous pouvez utiliser le caractère joker "*" afin de renommer toutes ou partie des configurations.

Exemples :

  • "*" prendra en compte toutes les configurations.
  • "Nom de la configuration à renommer" "début*" - "Nouveau nom" "DEBUT*" --> toutes les configurations seront renommées en remplaçant le début du nom début par DEBUT.
  • "Nom de la configuration à renommer" "*usiné" - "Nouveau nom" "*USINE" --> toutes les configurations seront renommées en remplaçant la fin du nom usiné par USINE.
  • "Nom de la configuration à renommer" "*11*" - "Nouveau nom" "*22*" --> toutes les configurations seront renommées en remplaçant dans le nom 11 par 22.


Vous pouvez utiliser les instructions suivantes :

<FileName>, pour afficher le nom du document.

$Nom de la propriété, pour afficher une valeur de propriété du document (si le nom de la propriété contient un espace, utiliser %20% pour remplacer cet espace).

<Configuration>, pour afficher le nom de la configuration active du moment.

<Date>, pour afficher la date.

<Time>, pour afficher l'heure.


RESPECTER LA CASSE


  • En générale la description d'une configuration prend le nom de la configuration. Par le choix dans le menu déroulant, il est également possible de "Renommer" ou "Ne pas renommer",  la description de la configuration.
